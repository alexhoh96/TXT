XML
 21. Создать внешний репозиторий c названием XML.
 22. Клонировать репозиторий XML на локальный компьютер. - git clone https://github.com/alexhoh96/XML.git
 23. Внутри локального XML создать файл “new.xml”. - cd XML 
                                                     touch new.xml
 24. Добавить файл под гит. - git add new.xml
 25. Закоммитить файл. - git commit -m "New file"
 26. Отправить файл на внешний GitHub репозиторий. - git push
 27. Отредактировать содержание файла “new.xml” - написать информацию о себе (ФИО, возраст, количество домашних животных, будущая желаемая зарплата). Всё написать в формате XML.
 cat >> new.xml
<?xml version="1.0"?>
<person>
<firstName>Alex</firstName>
<lastName>Khokhlov</lastName>
<age>26</age>
<pets>0</pets>
<FutureSalary>50000</FutureSalary>
</person>
 ctrl + C
 28. Отправить изменения на внешний репозиторий.
 git add new.xml
 git commit -m "new file"
 git push
 29. Создать файл preferences.xml - touch preferences.xml
 30. В файл preferences.xml добавить информацию о своих предпочтениях (Любимый фильм, любимый сериал, любимая еда, любимое время года, сторона которую хотели бы посетить) в формате XML.
 cat >> preferences.xml
  <Hobbies>
 <FavoriteMovie>Гарри Поттер</FavoriteMovie>
 <FavoriteTVShow>Нет одного самого любимого</FavoriteTVShow>
 <FavoriteFood>шашлык</FavoriteFood>
 <FavoriteTimeOfYear>summer</FavoriteTimeOfYear>
 </Hobbies>
 ctrl + C
 git add sklls.xml
 31. Создать файл sklls.xml добавить информацию о скиллах которые будут изучены на курсе в формате XML
 cat > sklls.xml
 <skills>
 <sklls>Теория тестирования, SQL, Postman, Git</sklls>
 </skills>
 ctrl + C
 git add preferences.xml
 32. Сделать коммит в одну строку.
 git commit -m "new 2 file"
 33. Отправить сразу 2 файла на внешний репозиторий.
 git push
 34. На веб интерфейсе создать файл bug_report.xml.
 35. Сделать Commit changes (сохранить) изменения на веб интерфейсе.
 36. На веб интерфейсе модифицировать файл bug_report.xml, добавить баг репорт в формате XML.
  <?xml version="1.0"?>
  <bug_report>
  <ID></ID>
  <Header></Header>
  <PlaySteps></PlaySteps>
  <ExpectedResult></ExpectedResult>
  <ActualResult></ActualResult>
  <Environment></Environment>
  <Priority></Priority>
  </bug_report>
 37. Сделать Commit changes (сохранить) изменения на веб интерфейсе.
 38. Синхронизировать внешний и локальный репозиторий XML - git pull
